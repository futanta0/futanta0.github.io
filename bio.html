Example 1: Exponential Distribution

If a product failure time (in hours) is assumed to follow an exponential distribution with failure rate λ = 0.00025, answer the following:

  1. Probability the product survives 10,000 hours.
  2. Probability the product survives the next 20,000 hours.
  3. Mean time to failure (MTTF).
  4. Median time to failure.
  5. Time by which 30% of products are expected to fail.
  6. Time by which 63.21% will fail.
R Code 
lambda <- 0.00025

## (i) Probability of surviving 10,000 hours
S.10000 <- exp(-lambda*10000)
S.10000

## (ii) Probability of surviving the next 20,000 hours
S.20000 <- exp(-lambda*20000)
S.20000

## (iii) Mean time to failure (MTTF)
MTTF <- 1/lambda
MTTF

## (iv) Median time to failure
t_0.5 <- (1/lambda)*log(2)
t_0.5

## (v) Time by which 30% of products fail
t_0.3 <- (-1/lambda)*log(1-0.3)
t_0.3

## (vi) Time by which 63.21% fail
t_0.6321 <- -log(1-0.6321)/lambda
t_0.6321

Example 2: Weibull Distribution

If a product failure time (in hours) is assumed to follow a Weibull distribution with scale parameter η = 4000 and shape parameter β = 1.50, find:

  1. Mean and variance of time to failure.
  2. Probability the product operates for 5000 h.
  3. Probability the product operates for another 2000 h after already operating for 5000 h.
R Code 
eta <- 4000
beta <- 1.50

## (i) Mean and variance of time to failure
mean.T <- eta*gamma(1+1/beta)
mean.T
var.T <- eta^2 * (gamma(1+2/beta) - (gamma(1+1/beta))^2)
var.T

## (ii) Probability of operating for 5000 hours
S.5000 <- exp(-(5000/eta)^beta)
S.5000

## (iii) Probability of operating another 2000 h after 5000 h
S.7000 <- exp(-(7000/eta)^beta)
S.2000_5000 <- S.7000/S.5000
S.2000_5000

Problem 1 — Shirts (Graphical LPP)

A garment manufacturer has a production line making two styles of shirts. Style I needs 100 g of cotton thread, 200 g of Dacron thread and 200 g of linen thread. Corresponding requirements of style II are 100 g, 300 g and 200 g. The net contributions are 100 Taka for style I and 120 Taka for style II. The available inventory of cotton thread, Dacron thread and linen thread are, respectively, 34 kg, 36 kg and 32 kg. The manufacturer wants to determine the number of each style to be produced with the given inventory. Formulate the LPP model and solve it by graphical method using R code.

Solution: Let x₁ = number of Style I shirts, x₂ = number of Style II shirts.

Maximize Z = 100x₁ + 120x₂

Subject to:

  • 100x₁ + 100x₂ ≤ 34,000 (Cotton)
  • 200x₁ + 300x₂ ≤ 36,000 (Dacron)
  • 200x₁ + 200x₂ ≤ 32,000 (Linen)
  • x₁, x₂ ≥ 0

The binding constraints (Linen and Dacron) intersect at x₁ = 120, x₂ = 40, giving a maximum profit Z = 16,800 Taka.

R Code 
library(ggplot2)

x <- seq(0,200,1)
cotton <- (34000 - 100*x) / 100
dacron <- (36000 - 200*x) / 300
linen  <- (32000 - 200*x) / 200

ggplot() +
  geom_line(aes(x, cotton), col="blue") +
  geom_line(aes(x, dacron), col="green") +
  geom_line(aes(x, linen), col="red") +
  xlim(0,200) + ylim(0,200)

x1 <- c(0,0,160,100,40)
x2 <- c(160,120,0,40,120)
Z  <- 100*x1 + 120*x2
i  <- which.max(Z)

cat("Optimal =", x1[i], x2[i], "\n")
cat("Maximum Profit =", Z[i])

Problem 2 — Feed Mix (Minimize Cost)

An animal feed company must produce 200 kg of a mixture consisting of ingredients A and B daily. A costs Rs. 3 per kg and B costs Rs. 8 per kg. Not more than 80 kg of A can be used and at least 60 kg of B must be used. The company wants to know how much of each ingredient should be used to minimize cost. Formulate the LPP.

Solution: Let x₁ = kg of ingredient A, x₂ = kg of ingredient B.

Minimize Z = 3x₁ + 8x₂

Subject to:

  • x₁ + x₂ = 200
  • x₁ ≤ 80
  • x₂ ≥ 60
  • x₁, x₂ ≥ 0

Since B costs more per kg, cost is minimized by using as much A as allowed: x₁ = 80, x₂ = 120, giving minimum cost Z = 1,200.

R Code 
library(lpSolve)

obj <- c(3,8)
con <- matrix(c(1,1,
                1,0,
                0,1), nrow=3, byrow=TRUE)
dir <- c("=","<=",">=")
rhs <- c(200,80,60)

sol <- lp("min", obj, con, dir, rhs)
sol$solution
sol$objval

Problem 3 — Three Products (Maximize Profit)

A factory manufactures three products A, B and C which involve lathe work, grinding and assembling. The cutting, grinding and assembling times required for one unit of A are 2, 1 and 1 hour respectively. Similarly they were 3, 1 and 3 hours for one unit of B and 1, 3 and 1 hours for one unit of C. The profits on A, B and C are Tk. 2, Tk. 2 and Tk. 4 per unit respectively. There are available 300 hours of lathe time, 300 hours of grinder time and 240 hours of assembly time. How many units of each product should be produced to maximize profit? Formulate the problem mathematically and solve using R code.

Solution: Let x₁, x₂, x₃ = units of products A, B, C.

Maximize Z = 2x₁ + 2x₂ + 4x₃

Subject to:

  • Lathe: 2x₁ + 3x₂ + x₃ ≤ 300
  • Grinding: x₁ + x₂ + 3x₃ ≤ 300
  • Assembly: x₁ + 3x₂ + x₃ ≤ 240
  • x₁, x₂, x₃ ≥ 0

Product A dominates B (same profit per unit but uses fewer resources everywhere), so B drops out (x₂ = 0). Solving the reduced two-variable LP gives the optimum at x₁ = 120, x₂ = 0, x₃ = 60, for a maximum profit Z = 480.

R Code 
library(lpSolve)

obj <- c(2,2,4)
con <- matrix(c(2,3,1,
                1,1,3,
                1,3,1), nrow=3, byrow=TRUE)
rhs <- c(300,300,240)
dir <- rep("<=",3)

sol <- lp("max", obj, con, dir, rhs)
sol$solution
sol$objval

Problem 4 — Simplex Method

Solve the following LPP using R code by Simplex method:

Minimize Z = x₁ + x₂ − 4x₃

Subject to:

  • x₁ + x₂ + 2x₃ ≤ 8
  • x₁ + x₂ − x₃ ≤ 2
  • −x₁ + x₂ + x₃ ≤ 4
  • x₁, x₂, x₃ ≥ 0

Solution: This is already in standard LP form. Since x₃ carries a negative objective coefficient (−4), the simplex method pushes x₃ as high as possible while keeping x₁ = x₂ = 0. That gives x₁ = 0, x₂ = 0, x₃ = 4, for a minimum Z = −16.

R Code 
library(lpSolve)

obj <- c(1,1,-4)

R Code 
costs  <- matrix(c(4,8,8, 16,24,16, 8,16,24), nrow=3, byrow=TRUE)
supply <- c(76,82,77)
demand <- c(72,102,41)

# North-West Corner Rule
nwcr <- function(supply, demand) {
  m <- length(supply); n <- length(demand)
  alloc <- matrix(0, m, n)
  i <- j <- 1
  while (i <= m && j <= n) {
    a <- min(supply[i], demand[j])
    alloc[i,j] <- a
    supply[i] <- supply[i] - a
    demand[j] <- demand[j] - a
    if (supply[i] == 0) i <- i + 1 else j <- j + 1
  }
  alloc
}

# Vogel's Approximation Method
vam <- function(cost, supply, demand) {
  m <- nrow(cost); n <- ncol(cost)
  alloc <- matrix(0, m, n)
  while (any(supply > 0) && any(demand > 0)) {
    pen <- rep(-Inf, m + n)
    for (i in which(supply > 0)) {
      row <- sort(cost[i, demand > 0])
      pen[i] <- if (length(row) > 1) row[2] - row[1] else row[1]
    }
    for (j in which(demand > 0)) {
      col <- sort(cost[supply > 0, j])
      pen[m + j] <- if (length(col) > 1) col[2] - col[1] else col[1]
    }
    k <- which.max(pen)
    if (k <= m) {
      i <- k; j <- which(demand > 0)[which.min(cost[i, demand > 0])]
    } else {
      j <- k - m; i <- which(supply > 0)[which.min(cost[supply > 0, j])]
    }
    a <- min(supply[i], demand[j])
    alloc[i,j] <- a
    supply[i] <- supply[i] - a
    demand[j] <- demand[j] - a
  }
  alloc
}

cat("NWCR Allocation:\n"); print(nwcr(supply, demand))
cat("\nVAM Allocation:\n");  print(vam(costs, supply, demand))
