Type 1: Design of Experiments (RBD / CRD)

Lab exercises of this type give you raw data from an experiment (e.g. diets fed to animals, or a randomized block layout) and ask you to test whether treatment means differ significantly.

  • You are given a data table (treatments as columns/rows, replicates or blocks).
  • You state H0 (all treatment means are equal) and H1 (at least one differs).
  • You run a one-way or two-way ANOVA in R and read the F-value and p-value.
  • You compare the p-value to alpha = 0.05 and write a conclusion.

Where solved: Experiment 1 (CRD) and Experiment 2 (RBD) in Part A below.

Type 2: Monte Carlo Integration (Control Variates)

These exercises ask you to estimate a definite integral using random sampling instead of calculus, and to reduce the variance of the estimate using a control variate.

  • You are given an integral such as the area under a curve on [0,1].
  • You generate random uniform numbers, evaluate the function, and average the results to estimate the integral.
  • You then use a control variate (a related function whose exact mean is known) to reduce variance.
  • You report the estimate, the true/reference value, and the variance reduction achieved.

Where solved: Part B, sections 1-3 (Control Variate methods).

Type 3: Random Number Generation (LCG, Mid-Square, Hit-or-Miss)

These exercises ask you to build your own pseudo-random number generator from scratch, rather than using R's built-in random functions.

  • You are given a seed and, for LCG methods, multiplier/increment/modulus constants.
  • You write the recurrence relation by hand in R (a loop, not a built-in function).
  • You generate a short sequence of numbers and show they lie between 0 and 1.
  • Hit-or-miss exercises use this generator to estimate an integral by counting points under a curve.

Where solved: Part B, sections 4-7 (Hit-or-Miss, Mid-Square, LCG, Combined LCG).

Type 4: Discrete Distribution Sampling

These exercises ask you to generate random values from a discrete probability distribution using the inverse-CDF (transformation) method.

  • You are given a distribution (Binomial, Poisson, or Geometric) with fixed parameters.
  • You generate a uniform random number U and map it onto the distribution using cumulative probabilities.
  • You repeat this many times and summarize the generated sample (table/frequency/mean).

Where solved: Part B, section 8 (f), (g), (h) - Binomial, Poisson, Geometric.

Type 5: Continuous Distribution Sampling

These exercises ask you to generate random values from a continuous probability distribution using transformation methods (inverse-CDF, Box-Muller, Marsaglia polar method).

  • You are given a distribution (Uniform, Exponential, Gamma, Normal, Chi-square) with fixed parameters.
  • You transform one or two uniform random numbers into a value from the target distribution using the relevant formula.
  • You generate a sample and summarize it (mean, spread, or a short printed vector).

Where solved: Part B, section 8 (a)-(e), (i) - Uniform, Exponential, Gamma, Normal (Box-Muller), Chi-square, Normal (Marsaglia).

1. Monte Carlo Integration with a Control Variate

Approximate the integral of e-x/(1+x²) from 0 to 1 using Monte Carlo integration with a control variate, and compare the result to simple Monte Carlo integration.

Note: No formal hypothesis test is required here. The aim is to estimate the integral and compare the variance of simple Monte Carlo with the control variate method.

R Code 
set.seed(1)
m <- 10000
u <- runif(m)
g <- exp(-u)/(1+u^2)              # function we want to integrate

# Simple Monte Carlo
theta_simple <- mean(g)
var_simple   <- var(g)/m

# Control variate: f(x) = 1/(1+x^2); true integral of f over [0,1] is pi/4
f <- 1/(1+u^2)
mu_f <- pi/4
b <- -cov(g,f)/var(f)
g_cv <- g + b*(f - mu_f)
theta_cv <- mean(g_cv)
var_cv   <- var(g_cv)/m

data.frame(Method = c("Simple MC","Control Variate"),
           Estimate = c(theta_simple, theta_cv),
           Variance = c(var_simple, var_cv))

Interpretation:

Both estimates should be close to the true value of about 0.5225. The control-variate estimate has a noticeably smaller variance than the simple Monte Carlo estimate, because f(x) tracks g(x) closely and removes some of the random noise.

2. Comparing Two Different Control Variates

Use the method of control variates to estimate θ = ∫01 g(x)dx with g(x) = e-x/(1+x²), using (a) control variate f1(x) = 1/(1+x²), (b) control variate f2(x) = x/(1+x²). (c) Compare both to simple Monte Carlo -- which achieves greater variance reduction?

Note: No formal hypothesis test is required here. We compare the estimates and choose the method with the smaller variance.

R Code 
set.seed(1)
m <- 10000
u <- runif(m)
g <- exp(-u)/(1+u^2)              # function we want to integrate

# (a) control variate f1
f1   <- 1/(1+u^2)
mu_f1 <- pi/4
b1   <- -cov(g,f1)/var(f1)
cv1  <- g + b1*(f1 - mu_f1)

# (b) control variate f2
f2    <- u/(1+u^2)
mu_f2 <- 0.5*log(2)     # true integral of x/(1+x^2) over [0,1]
b2    <- -cov(g,f2)/var(f2)
cv2   <- g + b2*(f2 - mu_f2)

# (c) compare
data.frame(Method   = c("Simple MC", "Control f1", "Control f2"),
           Estimate = c(mean(g), mean(cv1), mean(cv2)),
           Variance = c(var(g)/m, var(cv1)/m, var(cv2)/m))

Interpretation:

All three estimates should be close to the true value. Whichever control variate produces the smallest variance in the table gives the greatest variance reduction -- typically f1(x) works better here because its shape is closer to g(x) than f2(x) is.

3. Control Variate via Regression Approach

Approximate the integrals from Questions 1 and 2 using the control-variate-with-regression approach.

Note: No formal hypothesis test is required here. Regression is used to find the control variate coefficient and improve the Monte Carlo estimate.

R Code 
fit1 <- lm(g ~ f1)
c1 <- coef(fit1)[2]
theta_reg1 <- mean(g) - c1*(mean(f1) - mu_f1)

fit2 <- lm(g ~ f2)
c2 <- coef(fit2)[2]
theta_reg2 <- mean(g) - c2*(mean(f2) - mu_f2)

c(Regression_with_f1 = theta_reg1, Regression_with_f2 = theta_reg2)

Interpretation:

The regression approach automatically finds the best control-variate coefficient (the regression slope) instead of computing cov/var by hand, and gives essentially the same estimate as Question 2, confirming the manual calculation was correct.

4. Hit-or-Miss Method for Monte Carlo Integration

Using the hit-or-miss Monte Carlo method, estimate the integral of (sin(πx))² from 0 to 1.

Note: No formal hypothesis test is required here. The aim is to estimate the integral using the hit-or-miss simulation technique.

R Code 
set.seed(125)

hit_or_miss <- function(a, b, g, n){
  x <- runif(n, a, b)          # random x values in [a, b]
  k <- g(x)                    # function value at each x
  c <- max(k)                  # upper bound (height of enclosing rectangle)
  y <- runif(n, 0, c)          # random y values in [0, c]

  hit <- ifelse(y <= k, 1, 0)  # 1 = point falls under the curve ("hit")
  n_hit <- sum(hit)

  area <- c * (b - a) * n_hit / n
  area
}

g <- function(x) (sin(pi * x))^2

hit_or_miss(0, 1, g, 10000)

Interpretation:

The hit-or-miss method encloses the curve g(x) in a rectangle of height c = max(g(x)) and width (b-a), then estimates the area under the curve as (fraction of random points that fall below the curve) × (rectangle area). The estimate should come out close to the true value of this integral, which is 0.5.

5. Mid-Square Method for Random Number Generation

Using the mid-square method with seed = 9876, generate the first 5 random numbers.

Note: No formal hypothesis test is required here. This question only asks for the generated sequence of pseudo-random numbers.

R Code 
midsquare <- function(seed, n){
  k <- nchar(as.integer(seed))     # number of digits in the seed
  r <- k/2                         # half the digit count (middle digits to keep)
  z <- rep(0, n)
  temp <- seed

  for(i in 1:n){
    temp <- temp^2                     # square the number
    temp <- temp %% 10^(2*k - r)       # drop excess digits from the left
    temp <- floor(temp / 10^r)         # keep the middle digits
    z[i] <- temp
  }

  R <- z/10^k                          # scale to [0,1]
  data.frame(z, R)
}

midsquare(9876, 5)

Interpretation:

Each new number is generated by squaring the previous number and keeping only the middle digits. Dividing by 10^k (where k is the digit count of the seed) rescales each result to a value between 0 and 1, usable as a pseudo-random uniform number. The mid-square method is simple to compute by hand but can degenerate into short repeating cycles or zeros, which is why LCG-type methods are usually preferred in practice.

6. Linear Congruential Generator (LCG)

Using the linear congruential method, generate the first 3 random numbers with seed = 2985, a = 32, c = 3, m = 5821. Report numbers to 5 decimal places.

R Code 
lcg <- function(seed, a, c, m, n){
  x <- numeric(n)
  x[1] <- (a*seed + c) %% m
  for (i in 2:n) x[i] <- (a*x[i-1] + c) %% m
  round(x/m, 5)
}
lcg(seed = 2985, a = 32, c = 3, m = 5821, n = 3)

Interpretation:

Each output is x_i/m, scaled to lie between 0 and 1, so it can be used directly as a pseudo-random uniform number. The sequence is fully determined by the seed and parameters, so re-running with the same seed reproduces the same numbers.

7. Combined Linear Congruential Generator

For 32-bit computers, combine k=2 generators with a1=40014, m1=2147483563 and a2=40692, m2=2147483399, using seed = 73. Generate 10 random numbers.

R Code 
combined_lcg <- function(seed, a1, m1, a2, m2, n){
  x <- seed; y <- seed
  u <- numeric(n)
  for (i in 1:n) {
    x <- (a1*x) %% m1
    y <- (a2*y) %% m2
    z <- (x - y) %% (m1 - 1)
    u[i] <- if (z > 0) z/m1 else (m1-1)/m1
  }
  u
}
combined_lcg(seed = 73, a1 = 40014, m1 = 2147483563, a2 = 40692, m2 = 2147483399, n = 10)

Interpretation:

Combining two generators cancels out some of the patterns that a single LCG can have, giving a longer effective cycle and better-looking randomness than either generator alone.

8(a). Uniform(a=2, b=3)

Generate a random sample of size 1000 from Uniform(2, 3). Draw a histogram with a density curve. Check that the sample mean/variance agree with the theoretical mean/variance.

Note: No formal hypothesis test is required; every value in the interval is equally likely, so we only check agreement between sample and theoretical mean/variance. Theoretical mean = 2.5, theoretical variance ≈ 0.083.

R Code 
runif_custom <- function(size, a, b){
  x <- rep(0, size)
  for(i in 1:size){
    x[i] <- a + (b - a)*runif(1)
  }
  x
}

set.seed(100)
x <- runif_custom(1000, 2, 3)

hist(x, freq = FALSE, main = "Uniform(2, 3) sample")
lines(density(x), col = "blue", lwd = 2)

mean(x)
var(x)

Interpretation:

The theoretical mean of Uniform(a,b) is (a+b)/2 = 2.5, and the theoretical variance is (b-a)²/12 ≈ 0.083. The sample mean/variance should be close to these values, confirming the generator works correctly.

8(b). Exponential(rate = 2)

Generate a random sample from Exponential(rate = 2), draw a histogram with a density curve, check agreement with theoretical mean/variance, and test a hypothesis about the mean.

Hypotheses: H0: λ = 2 (equivalently μ = 0.5)   vs   H1: λ ≠ 2 (equivalently μ ≠ 0.5).

R Code 
rexpon <- function(size, lambda) -log(runif(size))/lambda

set.seed(100)
x <- rexpon(1000, 2)

c(sample_mean = mean(x), theoretical_mean = 1/2)
c(sample_var  = var(x),  theoretical_var  = 1/4)

hist(x, freq = FALSE, main = "Exponential(rate=2) sample")
lines(density(x), col = "blue", lwd = 2)

# H0: true mean = 0.5 (equivalent to H0: lambda = 2)
z <- (mean(x) - 0.5)/(sd(x)/sqrt(length(x)))
c(z = z, p_value = 2*(1-pnorm(abs(z))))

Interpretation:

The sample mean and variance should be close to 0.5 and 0.25, confirming the generator works. If the p-value is above 0.05, the data is consistent with rate = 2; if below 0.05, it isn't.

8(c). Gamma(shape = 3, rate = 2)

Generate a random sample from Gamma(shape = 3, rate = 2), draw a histogram with a density curve, check agreement with theoretical mean/variance, and test a hypothesis about the mean.

Hypotheses: H0: μ = 1.5   vs   H1: μ ≠ 1.5, since μ = α/λ = 3/2 = 1.5.

R Code 
rgam <- function(size, shape, rate){
  x <- numeric(size)
  for (i in 1:size) x[i] <- -sum(log(runif(shape)))/rate
  x
}

set.seed(100)
x <- rgam(1000, 3, 2)

c(sample_mean = mean(x), theoretical_mean = 3/2)
c(sample_var  = var(x),  theoretical_var  = 3/4)

hist(x, freq = FALSE, main = "Gamma(3,2) sample")
lines(density(x), col = "blue", lwd = 2)

z <- (mean(x) - 1.5)/(sd(x)/sqrt(length(x)))
c(z = z, p_value = 2*(1-pnorm(abs(z))))

Interpretation:

A Gamma(shape, rate) variate can be built as the sum of `shape` independent Exponential(rate) variates. The sample mean/variance should be close to 1.5 and 0.75; the p-value tells us whether the data still looks consistent with rate = 2.

8(d). Normal(mean = 5, sd = 2) via Box-Muller

Generate a random sample from Normal(mean = 5, sd = 2) using the Box-Muller transform, draw a histogram with a density curve, check agreement with theoretical mean/variance, and test a hypothesis about the mean.

Hypotheses: H0: μ = 5   vs   H1: μ ≠ 5.

R Code 
rnormal_bm <- function(size, mean, sd){
  n2 <- ceiling(size/2)
  x <- numeric(2*n2)
  for (i in 1:n2) {
    u1 <- runif(1); u2 <- runif(1)
    x[2*i-1] <- mean + sd*sqrt(-2*log(u1))*cos(2*pi*u2)
    x[2*i]   <- mean + sd*sqrt(-2*log(u1))*sin(2*pi*u2)
  }
  x[1:size]
}

set.seed(100)
x <- rnormal_bm(1000, 5, 2)

c(sample_mean = mean(x), theoretical_mean = 5)
c(sample_var  = var(x),  theoretical_var  = 4)

hist(x, freq = FALSE, main = "Normal(5,2) via Box-Muller")
lines(density(x), col = "blue", lwd = 2)

t.test(x, mu = 5)

Interpretation:

The Box-Muller algorithm turns two uniform numbers into two independent standard normal numbers, then rescales them to mean 5 and sd 2. If the t-test p-value is above 0.05, the sample mean is consistent with 5.

8(e). Chi-square(df = 10)

Generate a random sample from Chi-square(df = 10), draw a histogram with a density curve, check agreement with theoretical mean/variance, and test a hypothesis about the mean.

Hypotheses: H0: μ = 10   vs   H1: μ ≠ 10. For a chi-square distribution, the theoretical mean equals the degrees of freedom.

R Code 
chi <- function(size, df){
  x <- numeric(size)
  for (i in 1:size) x[i] <- -2*sum(log(runif(df/2)))
  x
}

set.seed(100)
x <- chi(1000, 10)

c(sample_mean = mean(x), theoretical_mean = 10)
c(sample_var  = var(x),  theoretical_var  = 20)

Generate a random sample from Binomial(n = 7, p = 0.6), draw a bar plot, check agreement with theoretical mean/variance, and test a hypothesis about p.

Hypotheses: H0: p = 0.6   vs   H1: p ≠ 0.6. Equivalently, H0: μ = np = 7(0.6) = 4.2.

R Code 
rbino <- function(size, n, p){
  X <- numeric(size)
  for (i in 1:size) {
    u <- runif(1); k <- 0
    prob <- (1-p)^n; cdf <- prob
    while (u > cdf) {
      k <- k + 1
      prob <- prob*(n-k+1)/k*p/(1-p)
      cdf <- cdf + prob
    }
    X[i] <- k
  }
  X
}

Pois <- function(size, lambda){
  y <- numeric(size)
  for (i in 1:size) {
    u <- runif(1); k <- 0
    prob <- exp(-lambda); cdf <- prob
    while (u > cdf) {
      k <- k + 1
      prob <- prob*lambda/k
      cdf <- cdf + prob
    }
    y[i] <- k
  }
  y
}

Hypotheses: H0: μ = 3   vs   H1: μ ≠ 3.

R Code 
mars <- function(size, mean, sd){
  x <- numeric(size)
  i <- 1
  while (i <= size) {
    u1 <- runif(1, -1, 1); u2 <- runif(1, -1, 1)
    w <- u1^2 + u2^2
    if (w < 1) {
      c0 <- sqrt(-2*log(w)/w)
      x[i] <- mean + sd*u1*c0
      if (i+1 <= size) x[i+1] <- mean + sd*u2*c0
      i <- i + 2
    }
  }
  x
}

set.seed(100)
x <- mars(1000, 3, 4)

c(sample_mean = mean(x), theoretical_mean = 3)
c(sample_var  = var(x),  theoretical_var  = 16)

hist(x, freq = FALSE, main = "Normal(3,4) via Marsaglia's Polar Method")
lines(density(x), col = "blue", lwd = 2)

t.test(x, mu = 3)

Interpretation:

Marsaglia's polar method avoids the sine/cosine calls used in Box-Muller by rejecting points outside the unit circle, which makes it slightly faster. The sample mean/variance should be close to 3 and 16, and the t-test tells us whether the sample mean is still consistent with 3.
